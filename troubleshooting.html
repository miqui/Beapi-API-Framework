Troubleshooting

I'm getting 404 when trying to access API from Javascript frontend

Make sure you are authenticated and sending authentication via AJAX properly (http://grails-plugins.github.io/grails-spring-security-core/docs/manual/guide/authentication.html#ajax. Also make sure that if you are sending a 'cross-domain' request to encode your session cookie for such.


Calls to http://localhost/api_v0.1/controller/action/1 fail or give a blank page

This is because you have a context set for your application. You need to call the api with the appContext (usually the app name) or set the app to a blank context within your Config.groovy environment...


          grails.app.context = "/"

Error : An association from the table 'blah' refers to an unmapped class

It is most likely that you are using multiple datasources. Make sure your mappings in the newly created domains go to the correct datasource.


ApiToolkit-Init asks for 'Person' and 'Role' but authentication is separated from application

The only classes that need this are the 'Hook' domain and the 'HookController'. If you have separated your authentication, you will have to manually handle these references in the application and change the id in the domain to Long instead of Person.
